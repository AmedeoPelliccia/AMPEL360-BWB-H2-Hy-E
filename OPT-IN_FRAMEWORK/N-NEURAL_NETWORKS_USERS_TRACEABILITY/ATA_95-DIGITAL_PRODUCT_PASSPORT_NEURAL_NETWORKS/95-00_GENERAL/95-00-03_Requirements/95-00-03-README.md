# 95-00-03 Requirements — Digital Product Passport for Neural Networks

## Overview

This folder contains the complete requirements framework for the **Digital Product Passport (DPP)** system under ATA Chapter 95 — Neural Networks & Digital Passport for the AMPEL360 BWB H₂ Hy-E aircraft.

### Purpose

The DPP requirements define **what** the system must do to:
- Support neural network lifecycle management from training to decommissioning
- Enable regulatory compliance (EU AI Act, EU DPP framework, aviation standards)
- Provide traceability and audit capabilities
- Integrate with toolchain automation and CAOS operations

## Traceability Philosophy

### Folder = Category

Folders represent **requirement categories** that align with DPP lifecycle concerns:
- Functional behavior
- Quality attributes
- Safety considerations
- Regulatory mandates
- Data governance
- Tooling integration
- Change management

### File = Atomic Requirement

Each file is **one atomic requirement** with:
- Its own version history in git
- Its own traceability links
- Its own acceptance criteria
- Its own verification evidence

### Tables = Summary Views

The `00_INDEX/` folder provides **aggregated views**:
- **Overview**: Narrative summary of requirements landscape
- **REQ_List.csv**: One row per requirement (ID, title, category, status)
- **RTM**: Requirements Traceability Matrix linking requirements to design, test, and certification

## Usage Guidelines

### Adding a New Requirement

1. Determine the appropriate **category** (01–07)
2. Assign the next available **ID** in that range
3. Create a new `.md` file using the naming convention:
   ```
   RQ-95-00-03-XXX_Short_Descriptive_Title.md
   ```
4. Fill in the template with complete information
5. Update `00_INDEX/95-00-03-REQ_List.csv` with the new requirement
6. Update RTM if applicable

### Modifying an Existing Requirement

1. Locate the requirement file by ID
2. Edit the `.md` file (git will track changes)
3. Update the **Status** section to reflect current state
4. If status changes to `APPROVED`, update the date
5. Update index files if title or category changes

### Tracing Requirements

- Use `grep -r "RQ-95-00-03-XXX"` to find all references to a requirement
- Check RTM for downstream dependencies (design, test, certification)
- Use MCP/doc-meta-enforcer to auto-link requirements across the repository
