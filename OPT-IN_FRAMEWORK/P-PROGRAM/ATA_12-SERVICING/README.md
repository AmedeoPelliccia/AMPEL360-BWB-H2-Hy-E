# ATA 12 — SERVICING

## Overview

This is ATA Chapter 12: SERVICING, part of the P — PROGRAM axis.

## Structure

This chapter follows the mandatory OPT-IN Framework structure:

### 12-00_GENERAL (Lifecycle Folders)

The GENERAL layer contains 14 mandatory lifecycle folders covering the complete development cycle:

1. **12-00-01_Overview**: System overview and global architecture
2. **12-00-02_Safety**: Safety framework and analysis
3. **12-00-03_Requirements**: Requirements and traceability
4. **12-00-04_Design**: Design specifications and patterns
5. **12-00-05_Interfaces**: Interface control documents
6. **12-00-06_Engineering**: Analysis, models, and simulation
7. **12-00-07_V_AND_V**: Verification and validation
8. **12-00-08_Prototyping**: Prototype development
9. **12-00-09_Production_Planning**: Manufacturing planning
10. **12-00-10_Certification**: Certification evidence
11. **12-00-11_EIS_Versions_Tags**: Configuration management
12. **12-00-12_Services**: Maintenance and service
13. **12-00-13_Subsystems_Components**: Component breakdown
14. **12-00-14_Ops_Std_Sustain**: Operational standards

### Cross-ATA Root Buckets

The following buckets are mandatory in every ATA chapter:

- **12-10_Operations**: Operational procedures and use cases
- **12-20_Subsystems**: Functional subsystems (design-driven internal structure)
- **12-30_Circularity**: Sustainability, LCA, and circular economy
- **12-40_Software**: Software, control logic, and AI/ML
- **12-50_Structures**: Physical structures and frames
- **12-60_Storages**: Tanks, reservoirs, and storage
- **12-70_Propulsion**: Propulsion interfaces (if applicable)
- **12-80_Energy**: Energy management and distribution
- **12-90_Tables_Schemas_Diagrams**: Data tables and documentation
