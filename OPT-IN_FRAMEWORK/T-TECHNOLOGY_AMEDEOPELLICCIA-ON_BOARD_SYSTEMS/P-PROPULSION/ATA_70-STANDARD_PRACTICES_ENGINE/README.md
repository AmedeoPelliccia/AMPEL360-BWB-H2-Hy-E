# ATA 70 — STANDARD PRACTICES ENGINE

## Overview

This is ATA Chapter 70: STANDARD PRACTICES ENGINE, part of the T — TECHNOLOGY AMEDEOPELLICCIA — ON BOARD SYSTEMS axis.

## Structure

This chapter follows the mandatory OPT-IN Framework structure:

### 70-00_GENERAL (Lifecycle Folders)

The GENERAL layer contains 14 mandatory lifecycle folders covering the complete development cycle:

1. **70-00-01_Overview**: System overview and global architecture
2. **70-00-02_Safety**: Safety framework and analysis
3. **70-00-03_Requirements**: Requirements and traceability
4. **70-00-04_Design**: Design specifications and patterns
5. **70-00-05_Interfaces**: Interface control documents
6. **70-00-06_Engineering**: Analysis, models, and simulation
7. **70-00-07_V_AND_V**: Verification and validation
8. **70-00-08_Prototyping**: Prototype development
9. **70-00-09_Production_Planning**: Manufacturing planning
10. **70-00-10_Certification**: Certification evidence
11. **70-00-11_EIS_Versions_Tags**: Configuration management
12. **70-00-12_Services**: Maintenance and service
13. **70-00-13_Subsystems_Components**: Component breakdown
14. **70-00-14_Ops_Std_Sustain**: Operational standards

### Cross-ATA Root Buckets

The following buckets are mandatory in every ATA chapter:

- **70-10_Operations**: Operational procedures and use cases
- **70-20_Subsystems**: Functional subsystems (design-driven internal structure)
- **70-30_Circularity**: Sustainability, LCA, and circular economy
- **70-40_Software**: Software, control logic, and AI/ML
- **70-50_Structures**: Physical structures and frames
- **70-60_Storages**: Tanks, reservoirs, and storage
- **70-70_Propulsion**: Propulsion interfaces (if applicable)
