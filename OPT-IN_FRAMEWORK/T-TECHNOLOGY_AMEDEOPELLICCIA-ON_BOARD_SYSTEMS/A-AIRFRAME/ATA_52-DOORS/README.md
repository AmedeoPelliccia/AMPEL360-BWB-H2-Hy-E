# ATA 52 — DOORS

## Overview

This is ATA Chapter 52: DOORS, part of the T — TECHNOLOGY AMEDEOPELLICCIA — ON BOARD SYSTEMS axis.

## Structure

This chapter follows the mandatory OPT-IN Framework structure:

### 52-00_GENERAL (Lifecycle Folders)

The GENERAL layer contains 14 mandatory lifecycle folders covering the complete development cycle:

1. **52-00-01_Overview**: System overview and global architecture
2. **52-00-02_Safety**: Safety framework and analysis
3. **52-00-03_Requirements**: Requirements and traceability
4. **52-00-04_Design**: Design specifications and patterns
5. **52-00-05_Interfaces**: Interface control documents
6. **52-00-06_Engineering**: Analysis, models, and simulation
7. **52-00-07_V_AND_V**: Verification and validation
8. **52-00-08_Prototyping**: Prototype development
9. **52-00-09_Production_Planning**: Manufacturing planning
10. **52-00-10_Certification**: Certification evidence
11. **52-00-11_EIS_Versions_Tags**: Configuration management
12. **52-00-12_Services**: Maintenance and service
13. **52-00-13_Subsystems_Components**: Component breakdown
14. **52-00-14_Ops_Std_Sustain**: Operational standards

### Cross-ATA Root Buckets

The following buckets are mandatory in every ATA chapter:

- **52-10_Operations**: Operational procedures and use cases
- **52-20_Subsystems**: Functional subsystems (design-driven internal structure)
- **52-30_Circularity**: Sustainability, LCA, and circular economy
- **52-40_Software**: Software, control logic, and AI/ML
- **52-50_Structures**: Physical structures and frames
- **52-60_Storages**: Tanks, reservoirs, and storage
- **52-70_Propulsion**: Propulsion interfaces (if applicable)
