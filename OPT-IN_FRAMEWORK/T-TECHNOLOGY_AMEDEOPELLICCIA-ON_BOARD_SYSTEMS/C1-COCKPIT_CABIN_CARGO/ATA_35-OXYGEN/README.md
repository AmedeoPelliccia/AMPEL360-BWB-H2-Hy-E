# ATA 35 — OXYGEN

## Overview

This is ATA Chapter 35: OXYGEN, part of the T — TECHNOLOGY AMEDEOPELLICCIA — ON BOARD SYSTEMS axis.

## Structure

This chapter follows the mandatory OPT-IN Framework structure:

### 35-00_GENERAL (Lifecycle Folders)

The GENERAL layer contains 14 mandatory lifecycle folders covering the complete development cycle:

1. **35-00-01_Overview**: System overview and global architecture
2. **35-00-02_Safety**: Safety framework and analysis
3. **35-00-03_Requirements**: Requirements and traceability
4. **35-00-04_Design**: Design specifications and patterns
5. **35-00-05_Interfaces**: Interface control documents
6. **35-00-06_Engineering**: Analysis, models, and simulation
7. **35-00-07_V_AND_V**: Verification and validation
8. **35-00-08_Prototyping**: Prototype development
9. **35-00-09_Production_Planning**: Manufacturing planning
10. **35-00-10_Certification**: Certification evidence
11. **35-00-11_EIS_Versions_Tags**: Configuration management
12. **35-00-12_Services**: Maintenance and service
13. **35-00-13_Subsystems_Components**: Component breakdown
14. **35-00-14_Ops_Std_Sustain**: Operational standards

### Cross-ATA Root Buckets

The following buckets are mandatory in every ATA chapter:

- **35-10_Operations**: Operational procedures and use cases
- **35-20_Subsystems**: Functional subsystems (design-driven internal structure)
- **35-30_Circularity**: Sustainability, LCA, and circular economy
- **35-40_Software**: Software, control logic, and AI/ML
- **35-50_Structures**: Physical structures and frames
- **35-60_Storages**: Tanks, reservoirs, and storage
- **35-70_Propulsion**: Propulsion interfaces (if applicable)
