# 85-00-05-005: Interface Versioning and Change Control

## 1. Purpose
This document defines the process for managing changes to interface definitions to ensure compatibility and safety.

## 2. Versioning Scheme
*   **Major Version (X.0):** Incompatible changes (e.g., physical connector geometry change).
*   **Minor Version (0.Y):** Backward-compatible changes (e.g., adding a data field).
*   **Patch Version (0.0.Z):** Editorial corrections.

## 3. Change Control Process
1.  **Change Request (CR):** Submitted via the project management system.
2.  **Impact Analysis:** Assess impact on [Cross ATA Matrix](./85-00-05-004_Interface_Matrix_Cross_ATA.md) and [Categories Map](./85-00-05-002_Interface_Categories_Map.md).
3.  **Review:** Technical review by affected stakeholders (Airport, Airline, OEM).
4.  **Approval:** Formal approval by the Chief Engineer / Infrastructure Lead.
5.  **Implementation:** Update documentation and assets.

## 4. Configuration Management
All interface control documents (ICDs) are configuration-controlled items. The master list is maintained in the [Master Interface Catalog](./MASTER/85-00-05-010_Master_Interface_Catalog.md).
