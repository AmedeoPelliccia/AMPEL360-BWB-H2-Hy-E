# 85-00-05-012: Interface Template Guidelines

## 1. Purpose
This document provides the standard template and guidelines for creating new Interface Control Documents (ICDs) within the ATA 85 framework.

## 2. Standard ICD Structure
Every ICD should contain the following sections:

1.  **Header:** Title, ID, Version, Date.
2.  **1. Introduction:** Purpose and Scope of the interface.
3.  **2. Applicable Documents:** References to standards (ISO, SAE) and other project docs.
4.  **3. Interface Description:**
    *   Physical characteristics (dimensions, materials).
    *   Electrical characteristics (voltage, current, pinout).
    *   Logical characteristics (protocol, data format).
5.  **4. Requirements:** Specific "shall" statements.
6.  **5. Verification:** How the interface is tested (Inspection, Analysis, Test).
7.  **6. Appendices:** Diagrams, drawings, pin maps.

## 3. Formatting
*   Use Markdown for text.
*   Link to assets in the `ASSETS` folder.
*   Use relative paths for links.
*   Ensure all tables have headers.
