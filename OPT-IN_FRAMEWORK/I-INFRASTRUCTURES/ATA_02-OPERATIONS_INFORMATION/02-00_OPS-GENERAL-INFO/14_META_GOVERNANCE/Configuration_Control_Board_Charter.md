# Configuration Control Board (CCB) Charter

**Document ID:** AMPEL360-02-00-00-META-CCB  
**Version:** 1.0.0  
**Effective Date:** 2025-09-01

## Purpose

Establish governance for changes to ATA 02 Operations Information documentation.

## Authority

The CCB has authority to:
- Approve/reject all changes to baselined documentation
- Prioritize change implementation
- Allocate resources for change implementation
- Escalate critical issues to executive management

## Membership

| Role | Representative | Authority |
|------|---------------|-----------|
| **Chair** | Director of Operations | Final decision authority |
| **Secretary** | Documentation Manager | Administrative, no vote |
| **Member** | Chief Engineer | Full voting rights |
| **Member** | Safety Manager | Full voting rights |
| **Member** | Certification Manager | Full voting rights |
| **Member** | CAOS System Lead | Full voting rights |
| **Advisor** | Regulatory Liaison | Advisory, no vote |

## Meeting Cadence

- **Regular Meetings:** Bi-weekly (every other Thursday)
- **Emergency Meetings:** As needed (24-hour notice)
- **Annual Review:** Full governance review in Q4

## Decision Process

1. Change request submitted minimum 5 days before CCB meeting
2. Technical review completed before CCB meeting
3. CCB reviews impact assessment
4. Vote taken (simple majority required)
5. Decision documented within 24 hours
6. Approved changes assigned implementation priority

## Change Classification

**Class A (Critical):** Safety-related, regulatory compliance, certification impact  
**Class B (Major):** System interface, procedure changes, CAOS integration  
**Class C (Minor):** Editorial, clarifications, formatting  

Class A and B require CCB approval. Class C can be delegated to Document Control.
