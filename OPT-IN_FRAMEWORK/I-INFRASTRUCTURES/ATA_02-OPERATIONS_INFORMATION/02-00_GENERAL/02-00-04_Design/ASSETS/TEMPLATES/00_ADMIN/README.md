# TEMPLATES — Generic Template Library (ATA 02-00-04)

## 1. Purpose

This directory contains **generic templates** used throughout
**ATA 02-00-04_Design** (Operations Information infrastructures), including:

- Requirements, specifications, reports and procedures
- Presentations for design reviews and management briefings
- Spreadsheets (BoMs, risk registers, configuration matrices, logs)
- Script / automation templates
- Techpub / S1000D data module and manual section templates

BIM / CAD-specific templates are managed separately in:

`ASSETS/INSTALLATIONS/BIM_CAD_MODELS/05_TEMPLATES/`

This folder provides a **single canonical source** for all other
template types.

## 3. Naming Convention

Templates use the following pattern:

```text
[ATA]-TPL-[Domain]-[TemplateName]-R[rev].[ext]
```

Where:

* **[ATA]** = `02-00-04`
* **TPL**  = template artifact
* **[Domain]**:

  * `DOC`  – textual documents (Markdown/Word)
  * `PRES` – presentations
  * `XLS`  – spreadsheets
  * `SCR`  – scripts
  * `TD`   – techpub / S1000D-related
* **[TemplateName]** = short, descriptive name in PascalCase or
  kebab-style with underscores, e.g.:

  * `Requirement`, `RequirementsSet`, `SystemSpecification`,
    `InstallationProcedure`, `DesignReport`, `TestReport`
  * `DesignReview`, `ManagementSummary`
  * `BOM`, `RiskRegister`, `ConfigMatrix`
  * `DataExtraction`, `ModelCheck`, `ReportGenerator`
  * `S1000D_DataModule`, `OPS_ManualSection`, `MaintTask`
* **R[rev]** = revision number (`R01`, `R02`, ...)
* **[ext]** = file extension (`md`, `docx`, `pptx`, `xlsx`, `py`, `xml`, ...)

**Examples**

* `02-00-04-TPL-DOC-Requirement-R01.md`
* `02-00-04-TPL-DOC-InstallationProcedure-R01.md`
* `02-00-04-TPL-XLS-BOM-R01.xlsx`
* `02-00-04-TPL-PRES-DesignReview-R01.pptx`
* `02-00-04-TPL-SCR-ReportGenerator-R01.py`
* `02-00-04-TPL-TD-S1000D_DataModule-R01.xml`

## 4. Global Template Index & Traceability

`02-00-04-TPL_Global_Index.csv` should track at least:

* `Template_ID`
* `File_Name`
* `Domain` (DOC, PRES, XLS, SCR, TD)
* `TemplateName`
* `Revision`
* `Folder_Path`
* `Intended_Use` (Requirements, Specs, Test, Review, Techpub, etc.)
* `Referenced_By` (links to guideline docs, checklists, etc.)
* `Status` (`WORKING`, `FOR_REVIEW`, `APPROVED`, `ARCHIVED`)

If needed, each major subfolder (e.g. `10_DOC_TEMPLATES`) can have its own
local index, referenced from the global one.

---

## 5. Lifecycle & Usage

1. When a team needs a new template:

   * First check if a suitable one exists here.
   * If not, create it under the appropriate subfolder using the naming
     convention and register it in `02-00-04-TPL_Global_Index.csv`.
2. When a template is **significantly revised**:

   * Bump `Rxx`.
   * Move the previous revision to `99_ARCHIVE/` under a dated folder:
     `02-00-04-TPL-ARCHIVE_YYYY-MM-DD_<tag>/`.
3. Project documents (requirements, reports, manuals, etc.) should **declare**
   which template version they are based on, when relevant.
